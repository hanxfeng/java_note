- # 概述

	- 在单台服务器的情况下，使用互斥锁可以实现线程一个一个执行，但在多台服务器组成集群时，传统的互斥锁就失去了作用，因为JVM中依靠锁监视器判断是否已经有线程获取到了互斥锁，但在服务器集群的情况下，a服务器的锁监视器无法监控到b服务器的线程，所以会出现互斥锁失效的现象，这时就需要使用分布式锁
	  
	- 分布式锁是在分布式系统或集群模式下多进程可见并且互相排斥的锁，分布式锁要满足互斥，高可用，高性能，安全性


- # 不同的分布式锁实现方法对比

|     | MySQL                                | Redis                                     | Zookeeper         |
| --- | ------------------------------------ | ----------------------------------------- | ----------------- |
| 互斥  | 利用MySQL本身的互斥锁机制                      | 基于setnx这样的互斥命令                            | 利用节点的唯一性和有序性实现互斥  |
| 高可用 | 好，受限于MySQL本身可用性，MySQL本身支持主从模式，因此可用性好 | 好，Redis支持主从和集群，因此可用性好                     | 好，支持集群            |
| 高性能 | 一般，受限与MySQL本身性能，差于Redis因此性能一般        | 好，Redis本身性能高                              | 一般，，差于Redis       |
| 安全性 | 断开连接时，自动释放锁                          | 安全性一般，如果服务宕机又没有设置TTL,就会出现死锁。需要记得设置TTL防止死锁 | 创建的是临时节点，断开连接自动释放 |


- # 基于 Redis 实现分布式锁

	- ## 获取锁
	
		- `setnx lock 1` // 利用 setnx 的特性确保只有一个线程能获取锁
		
		- `expire lock 10` // 确保即使后续服务器宕机也不会因为没有释放锁而进入死锁
		
			- 如果服务器在建立锁但还没添加 TTL 时宕机，那么会陷入死锁，因此以上两个操作必须同时成功同时失败，可以使用如下命令实现：
		
		- `set lock 1 nx ex 10` // nx 和 ex 是参数，nx 表示只有在不存在时才创建，ex 表示设置过期时间，单位为 s
	
	- ## 释放锁
	
		- `del lock` // 手动进行释放
	
	- ## 可能出现的问题
	
		- 假设线程1先执行业务，在执行业务时发生了阻塞，锁超时释放了，这时线程2再来执行业务，就能获取到锁，然后执行业务，在线程2执行业务时，线程1业务结束，释放锁，此时线程3就能再次获取到锁，出现两个业务同时进行的情况
		
			- 解决方法：在释放锁时，获取 value，判断是否一致

- # LUA 脚本

	- ## LUA 执行 Redis 命令的方法：
	
		- `redis.call('命令名称','key','其他参数')`
		  
		- 例如执行 `set name lihua ： redis.call('set', 'name', 'lihua')`
		
		- Redis 调用脚本方法：`eval "return redis.call('set', 'name', 'lihua')"
		
		- 其中，key 和 value 可以使用引用形式，例如：`redis.call('set', keys[1],argv[1]) name lihua`
	

- # java 分布式锁代码示例：

```java
public class SimpleRedisLock implements ILock {
	private String name;
	private StringRedisTemplate stringRedisTemplate;
	public SimpleRedisLock (String name, StringRedisTemplate stringRedisTemplate) {
		this.name = name;
		this.stringRedisTemplate = stringRedisTemplate;
	}
	private static final String KEY_PREFIX = "lock:";
	private static final String ID_PREFIX = UUID.randomUUID().toString(true)+"-";
	private static final DefaultRedisScript<Long> UNLOCK_SCRIPT;
	static {
		UNLOCK_SCRIPT = new DefaultRedisScript<>();
		// ClassPathResource 可以直接获取 Resource 目录下的文件
		UNLOCK_SCRIPT.setLocation(new ClassPathResource("unlock.lua"));
		UNLOCK_SCRIPT.setResultType(Long.class);
	}
	@Override
	// 尝试获取锁
	public boolean tryLock (long timeoutSec) {
		// 获取线程标识
		String threadId = ID_PREFIX +Thread.currentThread().getID();
		
		// 获取锁
		Boolean success = stringRedisTemplate.opsForValue()
				.setIfAbsent(KEY_PREFIX + name, threadId, timeoutSec,TimeUnit.SECONDS);
		return Bollean.TRUE.equals(success);
	}
	
	// 释放锁
	public void unlock() {
		stringRedisTemplate.execute(
			UNLOCK_SCRIPT,
			// keys参数要求传入一个列表，Collections.singletonList可以生成一个单元素列表
			Collections.singletonList(KEY_PREFIX + name)，
			threadId);
	}
}
```
