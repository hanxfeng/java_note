- 概述：
	- 存储和读取数据的解决方案
	- 用于读写文件中的数据
- 分类：
	- 按流的方向
		- 输入流（读取文件）
		- 输出流（写入文件）
	- 按操作文件类型：
		- 字节流（可以操作所有类型文件）
		- 字符流（只能操作纯文本文件）
	- ![[IO流体系.png]]
- FileOutputStream：
	- 步骤：
		- 创建对象：`FileOutputStream fos = new FileOutputStream(文件路径/file对象)`
			- 文件不存在时会创建新文件，但父路径需要存在
			- 默认采用覆写的方式写入数据，但在 close 之前使用多个 `FileOutputStream` 对象写入会在之前对象写入的数据后面续写
		- 写出数据：`fos.write(数据)`
			- write 方法写入的是整数，但写入文件的是整数在 ASII 表中对应的字母
		- 释放资源：`fos.close`
			- 为了解除资源占用
	- 写数据的三种方式：
		- 根据传入参数决定写入方式
			- 传入一个 int 整数：一次写一个字节数据
			- 传入一个 byte 类型的数组：一次写一个字节数组
			- 传入一个 byte 类型数组和两个 int 整数：
				- 一次写入一部分数组，第一个 int 表示起始索引，第二个 int 表示写入个数
		- 可以先新建一个字符串数组，在使用 `.getBytes()` 将字符串数组转为 byte 数组再传入
	- 换行写：
		- 在需要换行的地方写 windows：`\r\n` linux：`\n` mac：`\r` 即可
	- 续写：
		- `FileOutputStream fos = new FileOutputStream(文件路径/file对象,true)`
			- 多余的参数默认是 false，表示关闭续写，创建对象会清空文件
			- true 时表示打开续写，创建对象时不会清空文件
- FileInputStream：
	- 步骤：
		- 创建对象：`FileiutputStream fos = new FileOutputStream(文件路径/file对象)`
			- 如果文件不存在就会报错
		- 读取数据：`int b = fis.read();`
			- 一次读取一个字节数据，返回的数据是 int 类型，是字符在 ASII 表中对应的数字
			- 输出需要强转为 char 类型数据
			- 读取到文件末尾（比如一共3个字符，但读取四次），会返回 -1
			- 可以传入一个 byte 数组，读取时会尽量装满这个数组，此时返回值是一个 int 整数，表示读取了几个数据，读取的数据被储存在 byte 数组中。多次使用会覆盖数组中的数据，因此当最后只读取一个数据时， byte 数组中的数据只被覆盖一个，如果读取到空数据，那么 byte 数组中的数据就不会被替代，因此打印出来不会有变化
		- 释放资源：`fis.close();`
	- 循环读取：
		- `int b`
		- `while ((b = fis.read()) != -1) {System.out.println((char) b);}`
- 字符集：
	- GBK：
		- 概述：国标扩的拼音简写，包括常用的简繁中文和日韩文
		- 储存规则：
			- 汉字：
				- 以两个字节储存。一个字节是 8 位二进制
				- 高位字节二进制一定以 1 开头，转成十进制后是负数
			- 英文：
				- 以一个字节储存，兼容 ASII，不足 8 位在前面补 0 补到 8 位
	- Unicode：
		- UTF-16：使用 2-4个字节保存
		- UTF-32：固定使用 4 个字节保持
		- UTF-8：使用 1-4 个字节保持
			- 英文使用 1 个字节表示
			- 中文使用 3 个字节表示
			- ![[UTF-8编码方式.png]]
	- 乱码原因：
		- 未完整读取整个汉字
		- 编码与解码方式不同
- 编码与解码：
	- 编码：
		- 对 String 数据 使用`.getBytes();` 就会以默认方式进行编码，传入`"GBK","UTF-8"`等编码方式会按照传入的编码方式进行编码
	- 解码：
		- 在新建 String 对象时将 byte 数组输入会自动进行解码。`String str = new String(byte数组,编码方式)`
- 字符输入流
	- 概述：
		- 字符流可以理解为 字节流 + 字符集，在输入（读取）时，一次读一个字节，遇到中文时一次读多个字节，具体数量与使用字符集有关。在输出（写入）时，会按照指定编码方式编码后再写入文件
- FIleReader：
	- 概述：主要用于读取文本文件
	- 步骤：
		- 创建字符输入流对象：
			- `FileReader fr = new FileReader(字符串/File对象);`
		- 读取数据：
			- 空参：`fr.read();`
				- 返回 int 类型数据
				- 默认一次读取一个字节，但遇到汉字会读取多个字节
				- 读取后底层会自动进行解码并转为十进制，然后返回这个十进制数字
			- 带参：`fr.read(char[] c);`
				- 相当于将读取数据，解码，强转三步合并，直接将强转后的字符对象放入数组中
				- 
		- 释放资源：`fr.close();`

- BufferedInputStream/BufferedOutputStream：
	- 概述：底层自带 8192 的缓冲区提高性能
	- 创建：
		- `BufferedInputStream bis = new BufferedInputStream(new FileInputStream(File对象/路径字符串),缓冲区大小);`
		- `BufferedOutputStream bis = new BufferedOutputStream(new FileInputStream(File对象/路径字符串),缓冲区大小);`
			- 缓冲区大小默认 8192 可自定义，数据类型是 int
	- wirter 等方法与字符流，字节流基本相同
