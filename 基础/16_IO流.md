- 概述：
	- 存储和读取数据的解决方案
	- 用于读写文件中的数据
- 分类：
	- 按流的方向
		- 输入流（读取文件）
		- 输出流（写入文件）
	- 按操作文件类型：
		- 字节流（可以操作所有类型文件）
		- 字符流（只能操作纯文本文件）
	- ![[IO流体系.png]]
- FileOutputStream：
	- 步骤：
		- 创建对象：`FileOutputStream fos = new FileOutputStream(文件路径/file对象)`
			- 文件不存在时会创建新文件，但父路径需要存在
			- 默认采用覆写的方式写入数据，但在 close 之前使用多个 `FileOutputStream` 对象写入会在之前对象写入的数据后面续写
		- 写出数据：`fos.write(数据)`
			- write 方法写入的是整数，但写入文件的是整数在 ASII 表中对应的字母
		- 释放资源：`fos.close`
			- 为了解除资源占用
	- 写数据的三种方式：
		- 根据传入参数决定写入方式
			- 传入一个 int 整数：一次写一个字节数据
			- 传入一个 byte 类型的数组：一次写一个字节数组
			- 传入一个 byte 类型数组和两个 int 整数：
				- 一次写入一部分数组，第一个 int 表示起始索引，第二个 int 表示写入个数
		- 可以先新建一个字符串数组，在使用 `.getBytes()` 将字符串数组转为 byte 数组再传入
	- 换行写：
		- 在需要换行的地方写 windows：`\r\n` linux：`\n` mac：`\r` 即可
	- 续写：
		- `FileOutputStream fos = new FileOutputStream(文件路径/file对象,true)`
			- 多余的参数默认是 false，表示关闭续写，创建对象会清空文件
			- true 时表示打开续写，创建对象时不会清空文件
- FileInputStream：
	- 步骤：
		- 创建对象：`FileiutputStream fos = new FileOutputStream(文件路径/file对象)`
			- 如果文件不存在就会报错
		- 读取数据：`int b = fis.read();`
			- 一次读取一个字节数据，返回的数据是 int 类型，是字符在 ASII 表中对应的数字
			- 输出需要强转为 char 类型数据
			- 读取到文件末尾（比如一共3个字符，但读取四次），会返回 -1
			- 可以传入一个 byte 数组，读取时会尽量装满这个数组，此时返回值是一个 int 整数，表示读取了几个数据，读取的数据被储存在 byte 数组中。多次使用会覆盖数组中的数据，因此当最后只读取一个数据时， byte 数组中的数据只被覆盖一个，如果读取到空数据，那么 byte 数组中的数据就不会被替代，因此打印出来不会有变化
		- 释放资源：`fis.close();`
	- 循环读取：
		- `int b`
		- `while ((b = fis.read()) != -1) {System.out.println((char) b);}`
- 字符集：
	- GBK：
		- 概述：国标扩的拼音简写，包括常用的简繁中文和日韩文
		- 储存规则：
			- 汉字：
				- 以两个字节储存。一个字节是 8 位二进制
				- 高位字节二进制一定以 1 开头，转成十进制后是负数
			- 英文：
				- 以一个字节储存，兼容 ASII，不足 8 位在前面补 0 补到 8 位
	- Unicode：
		- UTF-16：使用 2-4个字节保存
		- UTF-32：固定使用 4 个字节保持
		- UTF-8：使用 1-4 个字节保持
			- 英文使用 1 个字节表示
			- 中文使用 3 个字节表示
			- ![[UTF-8编码方式.png]]
	- 乱码原因：
		- 未完整读取整个汉字
		- 编码与解码方式不同
- 编码与解码：
	- 编码：
		- 对 String 数据 使用`.getBytes();` 就会以默认方式进行编码，传入`"GBK","UTF-8"`等编码方式会按照传入的编码方式进行编码
	- 解码：
		- 在新建 String 对象时将 byte 数组输入会自动进行解码。`String str = new String(byte数组,编码方式)`
- 字符输入流
	- 概述：
		- 字符流可以理解为 字节流 + 字符集，在输入（读取）时，一次读一个字节，遇到中文时一次读多个字节，具体数量与使用字符集有关。在输出（写入）时，会按照指定编码方式编码后再写入文件
- FIleReader：
	- 概述：主要用于读取文本文件
	- 步骤：
		- 创建字符输入流对象：
			- `FileReader fr = new FileReader(字符串/File对象);`
				- jdk11 后可以传入一个 charset 对象来指定编码方式`charSet.forName("GBK")`
		- 读取数据：
			- 空参：`fr.read();`
				- 返回 int 类型数据
				- 默认一次读取一个字节，但遇到汉字会读取多个字节
				- 读取后底层会自动进行解码并转为十进制，然后返回这个十进制数字
			- 带参：`fr.read(char[] c);`
				- 相当于将读取数据，解码，强转三步合并，直接将强转后的字符对象放入数组中
				- 
		- 释放资源：`fr.close();`

- BufferedInputStream/BufferedOutputStream：
	- 概述：底层自带 8192 的缓冲区提高性能
	- 创建：
		- `BufferedInputStream bis = new BufferedInputStream(new FileInputStream(File对象/路径字符串),缓冲区大小);`
		- `BufferedOutputStream bis = new BufferedOutputStream(new FileInputStream(File对象/路径字符串),缓冲区大小);`
			- 缓冲区大小默认 8192 可自定义，数据类型是 int
	- wirter 等方法与字符流，字节流基本相同
- 字符缓冲流：
	- BufferedReader/BufferedWriter：
		- 构造方法与字节缓冲流相同
	- 特有方法：
		- `readline`：读取一行数据，如果没有数据可读，返回 null，不读取换行符
		- `newLine`：在不同系统实现相同的换行功能
- 转换流：
	- 概述：
		- 属于字符流，高级流的一种，分为 InputStreamReader 转换输入流和 OutputStreamWreter 转换输出流。是字节流和字符流之间的桥梁
		- 读取时将字节流转换为字符流，写入时将字符流转为字节流
		- 创建方式与缓冲流相同，可以额外传入字符串数据指定编码方式
	- 作用：使字节流可以使用字符流中的方法
- 序列化流 ObjectOutputStream：
	- 概述：可以将 java 中的对象写入本地文件中，高级流的一种，需要传入基本流
	- 构造方法：
		- ` ObjectOutputStream oos = new  ObjectOutputStream(new FileOutputStream(路径));`
	- 成员方法：
		- `oos.writerObject(写入对象)`
		- 使用 writerObject 方法需要在对象中实现 `Serializable` 接口，不需要实现这个接口中的方法，因为这个接口中没有方法，属于标记型接口，实现了这个接口表示当前类可以被序列化
- 反序列化流 ObjectinputStream：
	- 概述：将序列化到本地文件中的对象，读取到程序中来
	- 构造方法：
		- ` ObjectinputStream ois = new  ObjectinputStream(new FileinputStream(路径));`
	- 成员方法：
		- `Object o = ois.readObject();`
- 序列化流与反序列化流的注意事项：
	- 在将一个类写入文件的时候，会根据成员变量，构造方法，成员方法等生成一个序列号，如果对这个类进行修改，会改变这个序列号，这时再读入改变前报存的数据就会报错
	- 类完全不变实际情况下比较困难，可以在类中定义 `serialVersionUID` 来固定序列号`private static final long serialVersionUID = 序列号`
	- 不想将某个变量序列化到本地文件时可以加一个关键字 `transient`
- 字节打印流与字符打印流：
	- 概述：
		- 一般指 `PrintStream` 字节打印流和 `PrintWriter` 字符打印流
		- 打印流只操作文件目的地不操作数据源
		- 有特有的写出方法可以实现数据的原样写出，可以实现自动刷新，自动换行
	- 字节打印流：
		-  构造方法：
			- ![[字节打印流构造方法.png]]
			- 字节流底层没有缓冲区，自动刷新是否开启没有区别
			- 当自动刷新设置为 true 时，在调用 println，printf，format，写入换行符时，将缓冲区数据立刻写入到文件中
		- 成员方法：
			- ![[字节打印流成员方法.png]]
			- printf 可以使用 %s 进行占位，例：`printf("%s + %s","1","3");`，输出 1+3
	- 字符打印流：
		- 概述：
			- 与字节打印流相比，字符打印流底层多了缓冲区，效率更高，默认不开启自动刷新
		- 构造方法：`printWriter` 其他与字节打印流相同
		- 成员方法：与字节打印流相同
	- 额外补充：
		- 常用的`System.out`是系统中的标准输出流，在虚拟机启动的时候自动创建，默认指向控制台，在系统中唯一，关闭后除非重启虚拟机不然不能再打开。使用`PrintStream ps = System.out`创建
