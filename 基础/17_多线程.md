- 概述：
	- 线程：操作系统能够进行运算调度的最小单位，被包含在进程中，是进程中的实际运行单位
	- 进程：程序的基本执行实体
	- 并发：同一时刻，有多个指令在单个 cpu 上交替执行
	- 并行：同一时刻，有多个指令在多个 cpu 上同时执行
- 实现方式：
	1. 继承 Thread 类的方式实现
		1. 在创建类 MyThread 时继承 Thread 类
		2. 在 MyThread 中重写 run 方法
		3. 创建 MyThread 类的对象 t
		4. 使用 `t.start()` 启动线程，多线程就创建多个对象然后调用
		- ![[多线程第一种实现方法.png]]
	2. 实现 Runnable 接口的方式实现
		1. 创建类 MyRunnable 实现 Runnable 接口
		2. 在类中重写 run 方法
		3. 创建 MyRunnable 的对象 r 和 Thread 的对象 t
		4. 在创建 Thread 的对象 t 时将 r 传入
		5. 调用`t.start()` 方法
		- ![[多线程实现第二种方法.png]]
	3. 利用 Callable 和 Future 接口实现
		1. 创建一个类 MyCallabe 实现 Callable 接口
		2. 在类中重写 call 方法，有返回值
		3. 创建 MyCallabe 的对象 mc
		4. 创建 FutureTask 的对象 ft，`FutureTask<call方法的返回值> ft = new  FutureTask<>(mc);`
		5. 创建 Thread 的对象 t ，传入 ft
		6. 使用 `t.start();` 启动线程
		7. 获取多线程运行结果，`返回值类型 result = ft.get();`
		- ![[多线程实现第二种方法.png]]
- 常用实现方法：
	- ![[多线程常见成员方法.png]]
	- 未设置线程名称时，线程有默认名称，格式为 Thread-x ，x 是从 0 开始的序号
	- 需要给线程设置名字时，可以用 setName 方法设置，也可以在类中写构造方法，传入 String 类型的数据即可
	- currentThread 方法和 sleep 方法都是哪个线程执行到这个方法，哪个线程执行这个方法
	- 抢占式调度与非抢占式调度：
		- 线程的调度有两种方法，抢占式调度和非抢占式调度，抢占式调度是指哪条线程抢占到 cpu，哪条线程执行，具有随机性，非抢占式调度指所有线程轮流执行，java 中使用抢占式调度。
		- java 中优先级从 1-10 ，默认优先级是 5，越大抢占到 cpu 的概率越大
	- 守护线程：
		- 当其他非守护线程结束时，守护线程会陆续结束（不是一次性结束）
	- 礼让线程/出让线程：
		- 在执行到这个方法时，让出 cpu，一般写在线程中
	- 插入线程：
		- 将调用这个方法的线程插入到当前线程前先执行
- 线程的生命周期：
	- 线程的生命周期可以概况为以下四步：
		1. 新建：创建线程对象
		2. 就绪：程序已经就绪，处在抢 cpu 状态
		3. 运行：抢到 cpu 运行代码
		4. 死亡：线程执行结束，被销毁
		- 特殊：在运行时可能出现阻塞状态（sleep等方法造成），结束后转到就绪状态
- 同步代码块：
	- 概述：把操作共享数据的代码锁起来
	- 格式：`synchronized(锁){操作共享数据的代码}`
		- 锁默认打开，有一个线程进去，锁自动关闭
		- 锁中所有代码执行完毕，线程出来，锁自动打开
		- 锁对象一定是唯一的，即需要在前面加 static 关键字
		- 锁对象不一定要在操作共享数据的代码中出现
- 同步方法：
	- 概述：
		- 与同步代码块类似，同步方法是将整个方法都锁起来，在方法前加 synchronized 关键字即可
		- 同步方法的锁对象不能自己指定，由自己进行指定，非静态方法是 this，静态方法是当前类的字节码文件对象
- lock 锁：
	- 概述：lock 锁可以显式的获取和释放锁
	- 使用：
		- Lock 是一个接口，不能直接创建它的对象，一般是创建它的实现类 ReentrantLock 的对象：`Lock lock = new ReentrantLock();`
		- 加锁：`lock.lock();`
		- 释放锁：`lock.unlock();`
		- 如果需要多次创建锁所在类的对象，需要在创建时加上 static 关键字：`static Lock lock = new ReentrantLock();`
