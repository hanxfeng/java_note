
- # 概念：

	- 面向指使用，对象指工具，也可以理解为不同的 java 方法或 Python 库来解决问题，面向对象即是使用不同的方法进行编程，主要学习如何使用方法和如何编写方法
	
	- 特征：封装，继承，多态


- # 类和对象：

	- ## 概念：
	
		- 类像是图纸，对象则是具体的目标，可以认为对象是实例化的类
	
	- # 定义类：
	
		- `public class 类名 {};`
	
	- # 成员变量：
	
		- 按照标准不进行赋值，修饰符一般使用 private 进行私有化，后续使用 public 修饰的 set 方法来修改它的值
		
		- **定义格式**：`修饰符 数据类型 变量名称;`
	
	- ## 注意：
	
		- 一个文件中可定义多个类，但只有一个可以使用 public 进行修饰，其类名必须是文件名


- # 封装：

	- ## 原则：
	
		- 对象代表什么，就要封装对应数据（不用赋值），并提供数据对应行为
	
	- ## 标准：
	
		- 对应每一个成员变量，使用 private 进行私有化修饰，同时提供 set 方法以修改该变量的值，提供 get 方法来获取该变量，例：
```JAVA
		public class AddNumber {
			// 定义成员变量`
			private int a;
			private int b;
			
			// 定义 get 和 set 方法
			public void setA (int a) {
				this.a = a;
				}
				
			public void setB(int b) {
				this.b = c;
				}

			// 定义 get 方法
			public int getA() {
				return a;
				}

			// 定义 get 方法
			public int getB() {
				return b;
				}
		}
```


- # 就近原则：

	- ## 成员变量与局部变量
		
		- 定义在类中方法外的变量称为成员变量，定义在类的方法中的变量称为局部变量
	
	- ## 就近原则的概念：
	
		- 存在多个同名变量时，使用时遵循就近原则，即哪个变量离使用的地方更近，就使用哪个变量，不仅包括上文，也包括下文


- # 构造方法： 

	- ## 概述：
	
		- 也称为构造器或构造函数。用于在创建对象时给成员变量进行初始化
	
	- ## 格式：
	
		- `修饰符 类名/方法名（参数） {方法体};`
		
		- 方法名要与类名完全一致，包括大小写
		
		- 没有返回值类型，包括void，也没有返回值，不能使用return
	
	- ## 执行时机：
	
		- 创建对象时由 jvm 虚拟机自动调用，不能手动进行调用
		
		- 每创建一次对象，就会调用一次构造方法
		
		- 如果没有自定义构造方法，那么虚拟机会自动执行空参构造
	
	- ## 带参构造：
		```java
		public class AddNumber {
			private a;
			private b;
			};
	
			public AddNumber (int a, int b){
				this.a = a;
				this.b = b;
				}
			}
		```
		
		+ 此时便定义了一个带参构造，当使用`AddNumber an = new AddNumber(2,4)` 时便会自动给 a 和 b 进行赋值
		
		- 空参构造没有`int a,int b` 和方法中的方法体即可。
		
		- 如果定义了含参构造，那么也要定义无参构造
		
		- 标准讲至少要写一个无参构造和一个带全部参数的构造方法


- # 标准的 JavaBean 类：

	- ## 规则：
	
		- 1.类名要见名知意
		
		- 2.成员变量使用 private 修饰
		
		- 3.提供至少两个构造方法：无参构造与全参构造
		
		- 4.对每个成员变量提供 get 方法和 set 方法
	
	- 可以使用ptg插件快速生成 get 和 set 方法
	
	- 可用使用 Lombok 的@Data 注解快速生成两个构造方法和 get/set方法
