- # 静态变量与静态方法

	- ## 静态变量：
	
		- 被该类所有对象共享
		
		- 不属于对象，属于类
		
		- 随着类加载而加载，不随对象加载
	
	- ## 静态方法：
	
		- 多用于测试类和工具类，javabean 类中很少使用
		
		- 工具类：不是用来描述事物，而是帮我们做事的，比如 utils 包下的类
		
		- 测试类：顾名思义用于测试代码的类，带有 main 方法，是程序入口
		
		- javaBean 类：描述一些事务的类，比如 entity 层中的类
	
	- ## 注意事项：
	
		- 静态方法只能访问静态变量和静态方法
		
		- 非静态方法可以访问静态/非静态变量和方法
		
		- 静态方法中没有 this 关键字


- # 继承：

	- ## 概述
	
		- java 提供 extends 关键字，用于让一个类和另一个类建立继承关系，继承是指一个变量或方法在子类中没写，但子类仍然有并可以使用这个变量或方法
	
	- ## 使用时机：
	
		- 当类和类之间存在共同的内容，并且满足子类是父类的一种时，就可以考虑使用继承来优化代码
	
	- ## 特点：
	
		- java 只支持单继承，不支持多继承，但支持多层继承，每个类都直接间接继承与 Object。
		
		- 子类可以从父类中继承私有或非私有变量和非私有方法，不能继承私有或非私有构造方法和私有方法。[[7_面向对象#^02d166]]  父类的私有变量虽然能被子类继承，但是不能使用
		
		- **虚方法概念**：
		
			- 当创建一个类时，会创建一个虚方法表，这个表中包含：这个类的父类的虚方法表，这个类中的非 private，非 static，非 final 方法，依据虚方法表可以快速查询到父类中有的方法，虚方法表报存在方法区中
			
			- 不继承私有方法的原因：私有方法不会加入虚方法表中，因此不会被查询到，所以不被继承
	
	- ## 成员变量的访问特点：
	
		- **就近原则**：
			- 完整的就近原则，一个变量进行检索的顺序是本方法，本类，父类，如果都没有就会报错。
		
		- **访问**：
		
			- 可以使用 this 关键字访问本类的变量，使用 super 访问父类的变量
		
		- **重写父类方法**：
		
			- 在重写父类方法时，需要加 `@Override` 它的作用是检查子类的重写语法是否正确。
			
			- 重写方法后会覆盖虚方法表中的同名方法

			- 注意：
			
				- 重写方法的名称，形参列表要与父类一致
				
				- 子类重写方法时访问权限必须大于等于父类（不写<protected<public）
				
				- 返回值方法子类必须小于等于父类
				
				- 私有方法和静态方法不能重写，只有加入虚方法表的方法才能重写
	
	- ## 构造方法的继承特点：
	
		- 父类的构造方法不会被子类继承
		
		- 子类所有构造方法默认先访问父类的无参构造再执行自己
		（为了初始化父类中的数据）


- # 多态：

	- ## 概述：
	
		- 父类引用指向子类对象：`Animal a = new Dog();`
	
	- ## 前提：
	
		- 有继承关系
		
		- 有父类引用指向子类对象
		
		- 有方法的重写
	
	- ## 特点：
	
		- **变量调用：编译看左边，运行看左边**
		
			- 编译看左边指编译的时候会检查父类中有没有使用的变量，有才编译成功
			
			- 运行看左边指的是在 java 运行代码的时候，获取的是父类中该变量的值
			
			- 左右指等号左右
		
		- **方法调用：编译看左边，运行看右边**
		
			- 编译看左边指编译代码的时候会看父类中有没有这个方法，有才编译成功
			
			- 运行看右边指的是运行 java 代码的时候，运行的是子类中的这个方法
		
		- 在如果不使用多态的方式创建对象，那么在使用变量是就是使用子类的变量


- # 权限修饰符：

	- ## 概述：
	
		- 用于控制一个成员能被访问的范围，可以修饰成员变量/方法/构造方法/内部类
	
	- ## 不同修饰符：
	
		- **private**：只能在同一个类中使用
		
		- **空着不写**：能在同一个类和同一个包的其他类中使用
		
		- **protected**：能在同一个类、同一个包的其他类和不同包中的子类中使用
		
		- **public**：能在任何类中使用
	
	- ## 一般使用规则：
	
		- 成员变量私有，方法公开
		
		- 特例：如果方法中的代码是抽取的其他方法中的共性代码，一般也私有


- # 代码块：

	- ## 局部代码块：
	
		- **定义**：写在方法中的单独的大括号（不是定义方法的大括号）
		
		- **作用**：提前结束变量的生命周期，节约内存，现在不常用
	
	- ## 构造代码块：
	
		- **定义**：写在成员位置的代码块（构造方法前，成员变量后）
		
		- **作用**：将多个构造方法中重复的部分抽取出来，在执行构造方法前执行，不够灵活因此不常用
		
		- **代替方法**：
		
			- 将在多个构造方法中重复的代码写在一个构造方法在，其他构造方法中写 `this(其他构造方法参数)` 
			
			- 将重复代码定义成一个方法，在需要的地方执行这个方法
	
	- ## 静态代码块：
	
		- **概述**：通过 static 关键字修饰，随着类的加载而加载，自动触发，只执行一次
		
		- **使用场景**：在类加载时进行数据初始化


- # 包

	- ## 概述：
	
		- 包就是文件夹，用来管理不同的功能 java 文件
	
	- ## 书写规则：
	
		- 公司域名反写+包的作用
	
	- ## 全类名：
	
		- 包名+类名（com.hxf.add.AddNumber）
	
	+ ## 导入
	
		- 使用同一个包中的类时，不需要导入
		
		- 使用 java.lang 中的类时不需要导入
		
		- 使用在两个包中有相同类名的类时需要使用全类名


- # 抽象类：

	- ## 概述：
	
		- 将子类中共性的行为抽取到父类后，由于每一个子类执行的内容不一样，所以在父类中不能确定具体的方法体，该方法就可以定义为抽象方法，如果一个类中存在抽象方法，那么这个类必须声明为抽象类
	
	- ## 定义：
	
		- **抽象方法**：`public abstract 返回值类型 方法名 (参数);`
		
		- **抽象类**：`public abstract class 类名 ();`
	
	- ## 注意事项：
	
		- 抽象类不能实例化：
		
		- 抽象类中不一定有抽象方法，有抽象方法的一定是抽象类
		
		- 可以有构造方法
		
		- 抽象对象的子类必须重写所有抽象方法，或者该子类也是抽象类


- # 接口：

	- ## 概述：
	
		- 接口定义了一套标准规范，让不同的类能够以统一的方式被使用，实现程序的高度灵活性和扩展性。
	
	- ## 定义：
	
		- 使用 interface 关键字定义 `public interface 类名() {}`
		
		- 使用 `public class 类名 implements 接口名{};` 实现这个接口
		
		- 接口的实现类需要实现接口中所有的抽象方法，或者实现类也是抽象类
		
		- 一个实现类可以实现多个接口`public class 类名 implements 接口名1, 接口名2{};`
		
		- 实现类可以在继承一个类时实现多个接口`public class 类名 extends 父类名 implements 接口名1, 接口名2{};`
	
	- ## 接口中成员的特点：
	
		- **成员变量**：只能是常量，默认以 public static final 进行修饰
		
		- **构造方法**：无构造方法
		
		- **成员方法**：默认修饰符 public abstract ，jdk7 只能定义抽象方法，jdk8 可以定义有方法体的方法（包括默认方法和静态方法），jdk9 可以定义私有方法
		
	- ## 类和接口的关系：
	
		- **类和接口**：
		
			- 实现关系，可以一个类实现单个接口，也可以实现多个接口，还可以继承一个类的同时实现多个接口
		
		- **接口和接口**：
		
			- 继承关系，可以单继承也可以多继承
	
	- ## 在接口中定义有方法体的方法：
	
		- **作用**：
		
			- 可以在接口中定义方法后可以避免接口升级后实现该接口的所有类都需要重写接口中新增方法的问题
		
		- **默认方法格式**：
		
			- `public default 返回值类型 方法名(参数) {}`
			
			- 默认方法不强制进行重写，要重写的话重写时要去掉 default 关键字
			
			- `public` 可以省略 `default` 不行
			
			- 实现多个接口时，如果有重名方法，那么无论是否使用都要进行重写
		
		- ## 静态方法格式：
		
			- `public static 返回值类型 方法名(参数) {}`
			
			- 静态方法只能通过接口名调用，不能通过实现类名或对象名调用
			
			- `public` 可以省略 `static` 不行
		
		- ## 私有方法格式
		
			- **作用**：
			
				- 减少默认方法和静态方法中的重复代码，当多个方法中有重复代码时，就可以定义一个私有方法，来减少代码重复
			
			- **普通私有方法**：
			
				- `private 返回值类型 方法名(参数) {}`
			
			- **静态私有方法**：
			
				- `public static default 返回值类型 方法名(参数) {}`
			
			- 普通私有方法给默认方法服务，静态私有方法给静态方法服务
	
	- # 接口应用：
	
		- 接口代表规则，是行为的抽象，想让哪个类拥有一个行为，就让这个类实现对应的接口就可以
		
		- 当一个方法的参数是接口时，可以传递接口所有实现类的对象，这就是接口多态
	
	- ## 适配器设计模式：
	
		- 一个接口中有多个方法，如果只需要其中一个，又不想重写其他方法，就可以定义一个中间类，这个类是一个抽象类，对接口中所有方法进行空实现，再继承这个中间类，只重写需要的方法就可以了


- # 内部类：

	- ## 概述：
	
		- 写在一个类内部的类就是内部类。如果 b 是 a 的一部分，而 b 单独写又没意义，就可以将 b 定义为 a 的内部类 
	
	- ## 成员内部类：
	
		- **概述**：
		
			- 写在成员位置（变量与构造方法间），类中方法外，属于外部类成员，可以被修饰符修饰，此外，使用 static 修饰的是静态内部类，在成员内部类中，jkd16 之前不能定义静态变量，jdk16 后才可以
		
		- ## 创建对象方式：
		
			- 在外部类编写方法，对外提供内部类对象，类似 get 方法
			
				- `public 数据类型(内部类) 方法名() {return new 内部类名();}`
			
			- 直接创建
			
				- `外部类名.内部类名 对象名 = new 外部类对象名().new 内部类对象名();`
	
	- ## 静态内部类：
	
		- **概述**：
		
			- 静态内部类是成员内部类的一种特殊情况，使用 static 进行修饰，只能访问外部类的静态变量和静态方法，如果要访问非静态需要创建外部类对象
		
		- **创建对象方式**：
		
			-  `外部类名.内部类名 对象名 = new 外部类名.内部类名();`
		
		- **调用静态内部类的静态方法**：
		
			- `外部类名.内部类名.静态方法名`
		
		- **调用静态内部类的非静态方法**：
		
			- 先创建静态内部类对象，再进行调用
			
			- `外部类名.内部类名 对象名 = new 外部类名.内部类名();`
			
			- `对象名.非静态方法名();`
	
	- ## 局部内部类：
	
		- **概述**：
		
			- 将内部类定义在方法中的就是局部内部类，类似方法中的局部变量（[[7_面向对象#^2b6401]]）
			
			- 外界无法直接使用，需要在方法内部创建局部内部类的对象并使用
			
			- 该类可以直接访问外部类的成员，也可以访问方法内的局部变量
	
	- ## 匿名内部类：
	
		- **概述**：
		
			- 隐藏了名字的内部类，可以写在成员位置，也可以写在局部位置，它是一个类的子类对象或接口的实现类对象，包含了继承或实现，方法重写，创建对象
		
		- ## 格式：
		
			- `new 类名/接口名 () {重写方法};`  注意大括号后有分号
			
			- 包括三部分；
			
				- 实现/继承：`{重写方法}`
				
				- 方法重写：`{重写方法}`
				
				- 创建对象：`new`
			
			-  在大括号中实现/继承了一个接口类，并对其进行重写，然后使用 new 实现了这个类，小括号代表使用空参构造创建了一个对象，类指的是大括号部分
		
		- ## 使用场景：
		
			- 在进行测试等操作时，如果一个类只需要使用或临时使用一次，为此单独创建一个类比较麻烦，就可以使用匿名内部类，假设有一个方法，传入参数是一个类，方法的内容是调用类的一个方法：
			
			- `public void method(Animal a) {a.eat();}`
			
			- 这时就可以将匿名内部类作为参数输入，就会执行匿名内部类中重写后的 eat 方法
			
			- `new 类名/接口名 () {重写方法}.内部方法名();` 可以使用这种方式使用匿名内部类中重写的方法，也可以将这个匿名内部类赋值给变量，因为它输虽然叫匿名内部类，但实际上是一个对象


- # lambda 表达式：

	- ## 形式：
	
		- `(参数) -> {方法体}`
		
		- ()：对应方法的形参
		
		- -> ：固定格式
		
		- {}：对应方法的方法体
	
	- # 注意：
	
		- 只能用来简化匿名内部类书写
		
		- 只能简化函数式接口的匿名内部类的写法
		
			- 函数式接口：有且只有一个抽象方法的接口叫函数式接口，加@FunctionalInterface 进行注解
	
	- ## 省略规则：
	
		- 参数类型可以省略不写
		
		- 如果只有一个参数，参数类型可以省略，() 也可以省略
		
		- 如果方法体只有一行，大括号，分号，return 都可以省略，必须同时省略
