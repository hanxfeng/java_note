- 概述：
	- 代表程序出现的问题
	- ![[异常.png]]
	- java 中程序运行时出现的错误称为 exception 异常，error 错误是指系统级别的异常，与代码无关
- 作用：
	- 查询 bug 的关键参考信息
	- 作为方法内部的一种特殊返回值，以通知调用者底层的执行情况
- 异常处理方式：
	- jvm 默认处理方式：
		- 将异常名称，异常原因以及异常出现的位置等信息输出在控制台，异常下面的代码不再执行
	- 自己处理：
		- 格式：
			- `try{可能出现异常的代码} catch(异常类型) {异常的处理代码} final{}`
			- catch 可以有多个，如果异常类型存在父子关系，父类要写在子类下面，在 jdk7 之后，如果多种异常的处理方式相同，可以使用 `|` 将不同异常写在同一个 catch 中
			- 如果 try 中代码没有问题，会执行 try 中全部代码但不执行 catch，只有出现了异常才会执行 catch 中的代码
			- final 中的代码无论如何都会在 try 和 catch 中的代码执行后被执行
			- 如果 catch 中存在多种异常，同时有多个 catch 对应这些异常，那么之会执行第一个异常所对应的 catch ，因为异常被捕获后会跳转到对应 catch 中，后续代码不再执行
			- 如果 try 中遇到的问题没有被 catch 捕获，那么就会使用默认处理方式处理
		- 目的：代码出现异常时程序仍能继续运行
		- 常用方法：
			- ![[异常常用方法.png]]
	- 抛出异常：
		- throws：
			- 概述：写在方法定义处，表示声明一个异常告诉调用者，使用本方法可能出现哪些异常
			- 格式：`public void addNumber() throws 异常类名1,异常类名2{}`
				- 编译时异常必须要写，运行时异常可不写
		- throw：
			- 概述：写在方法内，结束方法手动抛出创建异常并返回给调用者，之后的代码不再执行
- 自定义异常：
	- 步骤：
		1. 定义异常类
			- 定义时后缀要加 Exception 表示这是一个异常类 
		2. 写继承关系
			- 运行时异常继承 RunTimeException
			- 编译时异常继承 Exception
		3. 写空参构造
		4. 写带参构造
			- 这两个构造可以不用自己写，在 idea 中按住 alt+ insert ，选择 Constructor，选择前两个点击 ok 即可
