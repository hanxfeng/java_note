	- ## Collection 遍历方式：
	
		- ### 迭代器遍历：
		
			- **概述**：
			
				- 是一个类，使用时需要创建对象使用，作用是将集合中的元素一个个遍历，不依赖索引
			
			- **实现和方法**：
			
				- 实现：`Iterator<String> it = 集合名.iterator;`
				
					- 创建时默认指针位置位于第一个元素
				
				- 判断是否存在元素：`boolean flag = it.hasNext();`
				
					- 如果迭代器当前指针指向位置存在元素则返回 true 否则返回 false
				
				- 获取元素并移动指针：`String str = it.next();`
				
					- 返回当前指针指向的元素并向后移动指针
				
				- 删除元素：`it.remove();`
				
					- 删除 next 返回的最后一个元素
				
			- **注意**：
			
				- 如果指针已经移动到集合末尾（没有元素的位置），再调用 next 方法就会报错
				
				- 指针在遍历至集合最后后不会复位，需要再创建一个对象
				
				- 迭代器遍历时不能使用集合的方式进行增加或删除，即在使用迭代器去遍历集合时不能同时对集合使用增加或删除操作，需要进行删除时使用 remove 方法进行删除
		
		- ### 增强 for 遍历：
		
			- **概述**：增强 for 底层就是迭代器，是为了简化迭代器代码书写的，只有单列集合和数组可以使用增强 for 进行遍历
			
			- **格式**：`for (元素的数据类型 变量名 : 数组或集合) {}`
			
				- 变量名表示变量时每次循环对应的数据
		
		- ### lambda 表达式遍历：
		
			- **格式**：`集合名.forEact(变量名 -> {语句})`
			
				- forEach 方法就是通过索引对集合进行遍历，完整的匿名内部类需要重写 accept 方法，变量传递给 accept 方法进行处理

- # 数据结构：

	- ## 概述：
	
		- 数据结构是计算机底层储存，组织数据的方式，指数据相互之间是以什么方式排列在一起的，数据结构是为了更加方便的管理和使用数据。
	
	- ## 栈：
	-
		- 特点：后进先出，先进后出。栈一端开口一端封闭
	
	- ## 队列：
	
		- 特点：先进先出，后进后出。队列两边开口。
	
	- ## 数组：
	
		- 特点：
		
			- 查询速度快，通过地址值和索引定位，查询任意数据耗时相同
			
			- 删除效率低，要将原数据删除同时将后续数据前移
			
			- 添加效率低，添加位置后每个元素都要往后移
	
	- ## 链表：
	
		- 概述：
		
			- 链表中的每一个元素称为结点，每个结点都是一个独立的对象，都有一个地址值，表示结点在内存中的位置，结点内存有具体数据和下一个结点的地址值
		
		- 特点：
		
			- 链表中的结点是独立对象，在内存中是不连续的，每个结点包含数据值和下一个结点的地址值。双向链表还包括前一个结点的地址值
			
			- 查询慢，无论查找哪个数据都有从头开始找
			
			- 增删快，对数据增删只要操作很少一部分就可以
	
	- ## 二叉树：
	
		- ### 概述：
		
			- 在二叉树中，每个节点下最多只有两个节点，分别是左子节点和右子节点，它们的上一个节点是父节点
			
			- 节点中存有数据，父节点地址值，左子节点地址值，右子节点地址值
			
			- 每个节点子节点的数量叫做度，树的高度是这个树的总层数，最上面的节点是根节点
		
		- ### 二叉查找树：
		
			- 概述：
			
				- 如果一个二叉树，它的每一个节点的左子节点比自己小，右子节点比自己大，这样的二叉树就被称为二叉查找树
			
			- 遍历方式：
			
				- 前序遍历：从根节点开始，按当前节点，左子节点，右子节点的顺序进行遍历
				
				- 中序遍历：从最左边的子节点开始，按照左子节点，当前节点，右子节点的顺序进行遍历
				
				- 后序遍历：从最左边的子节点开始，按照左子节点，右子节点，当前节点的顺序进行遍历
				
				- 层序遍历：从根节点开始一层一层遍历
		
		- ### 平衡二叉树：
		
			- 概述：
			
				- 在二叉查找树的基础上，添加了一个规则，任意节点左右子树高差不超过1
			
			- 旋转机制：
			
				- 当添加一个节点后，该树不是一个平衡二叉树，就会触发旋转机制。分为左旋和右旋
				
				- 首先要确定支点，从添加的节点开始，不断的往父节点寻找不平衡的节点，即左右子树层数差超过2的节点，这个节点就是支点。
				
				- 如果这个节点右边层数多，那么进行左旋，否则进行右旋
				
				- 左旋就是将支点降级，成为它右子节点的左子节点，它的右子节点则是代替它的位置
		
		- ### 红黑树：
		
			- 概述：
			
				- 一种自平衡的二叉查找树，红黑树的每一个节点上有储存位表示节点颜色，每个节点是红色或黑色，它不是高度平衡的，它的平衡通过红黑规则实现
			
			- 红黑规则：
			
				- 每一个节点不是红色就是黑色
				
				- 根节点必须是黑色
				
				- 如果一个节点没有子节点或父节点，则该节点相应的指针属性值为 nil ，这些 nil 视为叶节点，每个叶节点都是黑色
				
				- 如果某个节点是红色，那么它的子节点必须是黑色
				
				- 对每一个节点，该节点到其所有后代节点的简单路径上，均包含数量相同的黑色节点
			
			- 添加规则：
			
				- ![[红黑树添加规则.png]]
	
	- ### 哈希表：
	
		- 概述：哈希表是一种对于增删改查数据性能都比较好的数据结构
		
		- 组成：数组+链表+红黑树
		
		- 哈希值：
		
			- 概述：
			
				- 对象的整数表现形式，根据 hashCode 算法算出的 int 类型的整数，该方法定义在 Object 类中，所有对象都可以调用，使用默认地址值计算，一般情况下会重写 hashCode 方法，使用对象内部属性值计算
			
			- 特点：
			
				- 如果没有重写 hashCode 算法，不同对象计算出的哈希值不同
				- 重写后只要属性值相同计算出的哈希值就相同
			
			- 底层原理：
			
				1. 创建一个默认长度为 16，默认加载因子为 0.75 的数组，名字是 table，默认加载因子指数组使用多少后进行扩容，扩容成原来的两倍
				
				2. 根据元素哈希值与数组长度计算出应存入位置，计算方法：`数组长度-1 & 哈希值`
				
				3. 判断当前位置是否为 null ，如上是直接存入
				
				4. 如果不为 null，调用 equals 方法比较属性值，一样不存入数组形成链表，将新元素挂在老元素下
				
				5. 当链表长度大于8，数组长度大于64，链表会转为红黑树


- # ArrayList 集合：

	- ## 概述：
	
		- 利用空参构造创建集合时，会在底层创建一个默认长度为 0 的数组 elementData

- # 泛型：

	- ## 概述：
	
		- 可以在编译阶段约束操作的数据类型并进行检查，只支持引用数据类型
		
		- 指定泛型的具体类型后，传递数据时，可以传入该类类型或子类类型
		
		- 如果不写泛型，类型默认是 Object
		
		- 泛型还可以写在类，方法，接口后面
	
	- ## 格式：`<数据类型>`
	
	- ## 作用：
	
		- 统一集合中的数据类型
	
	- ## 泛型类：
	
		- 格式：`修饰符 class 类名<类型> {}`
		
		- 使用场景：当一个类中，某一个变量的数据类型不确定时，就可以定义带有泛型的类
		
			- 类型出写 E 可以理解为变量，实际上可以写任意字母，不过 T，E，K，V使用较多
	
	- ## 泛型方法：
	
		- 概述：当方法中形参不确定时，可以使用定义在类上的泛型，也可以使用定义在方法上的类型，区别是类上的泛型所有方法都可以使用，方法上的泛型只有本方法中可以使用
		
		- 定义：`修饰符<类型> 返回值类型 方法名 (类型 变量名) {}` 
	
	- ## 泛型接口
	
		- 格式：`修饰符 interface 接口名<类型> {}`
		
		- 使用：
		
			- 1.实现类给出具体类型
			
				- 在实现该接口时，将类型 E 换成具体数据类型即可
			
			- 2.实现类延续泛型，创建对象时再确定
			
				- 在实现接口时仍然使用 E 在创建对象时明确使用的数据类型即可
	
	- ## 泛型的继承和通配符
	
		- ### 泛型的继承：
		
			- 概述：
			
				- 泛型类型不具备继承性，如果类 B 是类 A 的子类，但容器类`G<B>`并不是`G<A>`的子类。因此，如果一个方法参数是`G<A>`类型，那么传入`G<B>`类型的对象是不允许的。但是，对于普通继承关系，子类对象可以之间赋值给父类引用
		
		- ### 通配符：
		
			- 概述：
			
				- 在 java 中，通配符是`?` ，表示不确定类型
				
				- 当在泛型的类型处使用`?`时代表可以传入任何数据
				
				- 当在泛型的类型处使用`? extends 类名/接口名` 时，表示可以传入类或接口的所有子类
				
				- 当在泛型的类型处使用`? super 类名/接口名` 时，表示可以传入类或接口的所有父类


- # Set 集合：

	- ## 特点：
	
		- 无序，不重复，无索引
	
	- ## 遍历方式：
	
		- 迭代器，增强 for 循环，Lambda 表达式
	
	- ## 实现类的特定：
	
		- HashSet：无序，不重复，无索引
		
		- LinkHashSet：有序，不重复，无索引
		
		- TreeSet：可排序，不重复，无索引


- # HashSet：

	- ## 概述：
	
		- HashSet 使用哈希表存储数据
	
	- ## 底层原理：
	
		1. 创建一个默认长度为 16，默认加载因子为 0.75 的数组，名字是 table，默认加载因子指数组使用多少后进行扩容，扩容成原来的两倍
		
		2. 根据元素哈希值与数组长度计算出应存入位置的索引，计算方法：`数组长度-1 & 哈希值`
		
		3. 判断当前位置是否为 null ，如上是直接存入
		
		4. 如果不为 null，调用 equals 方法比较属性值，一样不存入，不一样则将新元素挂在老元素下形成链表
		
		5. 当链表长度大于8，数组长度大于64，链表会转为红黑树


- # LinkHashSet：

	- LinkHashSet 会保证存入和取出顺序一致
	
	- 底层原理：仍是哈希表，不过每个元素额外多了双链表记录位置
	
	- 去重时默认使用 hashSet，要求去重且存取有序才使用 LinkHashSet

- # TreeSet：

	- ## 特点：
	
		- 不重复，无索引，可排序。
	
	- ## 概述：
	
		- 基于红黑树实现，增删改查性能都较为良好
	
	- ## 排序方式：
	
		- ### 默认：
		
			- 对于数值类型，默认按照从小到大的顺序进行排序
			
			- 对于字符，字符串类型，按照 ASCII码表的数字升序排序
		
		- ### 存储对象比较复杂，有多种数据时，通过 JavaBean 类实现 Comparable 接口指定比较规则：
		
			- 使用 Student 对象的 age 属性进行排序
				```java
				public int compareTo(Student o) {
					return this.getAge() - o.getAge();
					}
				```
			
			- this 表示要添加的元素，o 表示红黑树已存在的元素。
			
			- 返回值是负数时，认为要添加的元素是小的，存在左边
			
			- 返回值是正数时，认为要添加的元素是大的，存在右边
			
			- 返回值是0时，认为要添加的元素重复，舍弃
			
			- 当红黑树较大时，需要多次调用进行对比才能确定存放位置
		
		- ### 比较器排序：
		
			示例：
			- ![[示例1.png]]
			
			- 在创建集合时传入一个实现了 Comparator 接口的匿名内部类即可，这个匿名内部类需要重写 compare 方法，在这个方法中进行自定义排序
			
			- 返回值与第二种相同
		
		- 默认使用 1，2 种，不能满足需求时才使用比较器排序


 - # Map 集合：
 
	- ## 概述：
	
		- Map 是双列集合的顶层接口，它的功能全部双列集合都可以使用
		
		- 在创建时需要同时指定键的类型和值的类型，例：`Map(String,int) m = new hashMap<>;`
	
	- ## 方法：
	
		- ![[Map方法.png]]
		
		- `Set .keySet();` ：将所有键返回到一个 Set 集合中
		
		- `<E> .get(键);` ：通过输入的键获取集合中键所对应的值
		
		- put：在添加数据的时候，如果键不存在，那么直接将数据添加进去。如果键是存在的，那么会覆盖原有键值对，然后返回被覆盖的**值**
	
	- ## 遍历：
	
		- ### 键找值：
		
			- 使用 `keySet()` 方法获取所有键，然后再通过 `get()` 方法获取所有值
		
		- ### 依次获取所有键值对对象：
		
			- 概述：先获取所有键值对对象，再通过 getKey 方法获取所有键，通过 getValue 方法获取所有值
			
			- 语法：`Set<Map.Entry<键类型,值类型>> en = map.entrySet();`
				- 可以导入 Entry 然后就不用写 Map
		
		- ### Lambda 表达式：
		
			- 语法：`map.forEach(Lambda表达式)`


- # HashMap：

	- ## 特点：
	
		- 是 Map 的一个实现类
		
		- 没有额外的方法，直接使用 Map 的方法即可
		
		- 无序，不重复，无索引
		
		- HashMap 与 HashSet 底层原理都是哈希表
		
		- 使用键而不是值计算哈希值
		
		- 与 HashSet 不同的是，如果哈希值相同，那么后存入的数据会覆盖先存入的数据
		
		- 通过 hashCode 方法和 equals 方法保证键唯一
		
		- 键储存自定义对象需要重写 hashCode 方法和 equals 方法，值则不需要


- # LinkHashMap：

	- ## 特点：
	
		- 有序，不重复，无索引
		
		- 有序指存入和取出顺序一致
		
		- 与 LinkHashSet 一致，都是通过添加双链表确保有序


- # TreeMap：

	- TreeMap 与 TreeSet 一样都是红黑树
	
	- 不重复，无索引，可排序（对键进行排序）
	
	-  默认按照键的大小进行排序，也可以自己重写 compareTo 方法自定义排序规则，也可以创建集合时传入 Comparator 比较器对象指定比较规则


- # HashMap源码分析：

	- ## 前置知识：
	
		- `Node<K,V>[] table` 哈希表结构中数组的名字
		
		- `DEFAULT_INITIAL_CAPACITY` 数组默认长度，16
		
		- `DEFAULT_LOAD_FACTOR` 默认加载因子，0.75
		
		- ### HashMap 中每一个对象包括以下内容：
		
			- #### 链表中的键值对对象：
			
				- `int hash` 键的哈希值
				
				- `final K key` 键
				
				- `V value` 值
				
				- `Node<K,V> next` 下一个节点的地址值
			
			- #### 红黑树中的键值对对象：
				- `int hash` 键的哈希值
				
				- `final K key` 键
				
				- `V value` 值
				
				- `TreeNode<k,V> parent` 父节点的地址值
				
				- `TreeNode<k,V> left` 左子节点的地址值
				
				- `TreeNode<K,V> right` 右子节点的地址值
				
				- `boolean red` 节点的颜色，true 为红，false 为黑
			
			- #### 数组中的键值对对象：
			
				- 根据情况决定，如果是链表的开头，则与链表中的键值对对象相同，如果是红黑树父节点，则与红黑树中的键值对对象相同
	
	- ## put 源码分析
	
		1. 调用 `put(K key, V value)` 方法，传入键值对
		
		2. `put` 方法调用 `putVal` 方法，计算 key 的哈希值：`(key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16)`
		
		3. `putVal` 方法首先检查 table 数组是否为空或长度为 0：
		    - 如果是，调用 `resize()` 方法初始化或扩容数组
		
		4. 根据哈希值计算数组下标，检查该位置是否为空：
		
		    - 如果为空，直接创建新节点放入该位置
		    
		    - 如果不为空，处理哈希冲突：
		    
		        - 如果该位置节点的 key 与传入 key 相同（hash 相等且 key equals），记录该节点用于后续覆盖
		        
		        - 如果该位置节点是树节点，调用红黑树的插入方法
		        
		        - 如果是链表节点，遍历链表：
		        
		            - 找到相同 key 的节点，记录并跳出循环
		            
		            - 到达链表末尾仍未找到，创建新节点添加到链表尾部，检查链表长度是否达到树化阈值 8：
		            
		                - 如果达到，调用 `treeifyBin` 方法将链表转为红黑树
		    
		5. 如果存在相同 key 的节点（覆盖情况）：
		
			- 保存旧值，用新值替换旧值
			
		    - 返回被覆盖的旧值
        
		6. 如果是新增节点：
		
		    - 增加 modCount 记录结构修改次数
		    
		    - 检查元素数量是否超过扩容阈值（容量 * 加载因子）：
		    
		        - 如果超过，调用 `resize()` 方法进行扩容
		    
		    - 返回 null 表示没有覆盖任何元素
		
		- 注：实际 `resize()` 扩容逻辑为：
		
			- 创建新数组，长度为原数组的两倍
			
			- 重新计算所有元素在新数组中的位置并转移
			
			- 更新扩容阈值


- # 可变参数：

	- 概述：方法形参的个数可以变化
	
	- 格式：`类型...变量名`
	
	- 原理：java 底层会创建一个数组，名称与变量名相同，数组中就是传入的数据
	
	- 注意：
	
		- 形参中最多只有一个可变参数
		
		- 如果除了可变参数还有其他形参，那么可变参数必须写在最后
