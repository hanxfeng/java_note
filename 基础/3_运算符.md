- # 运算符：

	- ## 隐式转换：
	
		- java 的数据类型较多，不同数据类型没法直接进行运算，比如同属整型的 int 和 byte 就无法直接进行运算，如果要进行计算会将范围小的数据类型转为范围大的数据类型进行计算，java自动进行，byte short char 三种类型的数据在运算时会先提升为 int 再进行运算。
	
	- ## 强制转换：
	
		- 格式：`目标数据类型 变量名 = (目标数据类型) 被转换的变量` `int b = (int) a;`
		
		- 不能对超过目标数据类型范围的数据进行强制转换
	
	- ## 字符串与字符计算：
	
		- ## 字符相加：
		
			- 在 ASCII 表中有的字符进行计算时会转为对应数字，注意，特指字符而不是字符串
		
		- ## 字符串相加：
		
			- 字符串只有+运算，与任何数据进行运算时会进行拼接操作，如 123+"你好" = “123你好”
	
	- ## 自增自减运算符：
	
		- ++：
		
			- 使用方法：`a++;`
			
				- 相当于 `a = a+1;`
		
		- -- ：与 ++ 相同
	
	- ## 赋值运算符：
	
		- `+=，-=，*=，/=等：`
		
			- 与 python 相同
		
		- %=:
		
			- 取余后赋值，`a %= b` 将a/b的余数赋值给a
		
		- `==`：
		
			- 使用 == 进行比较有两种情况：
				- 1.基本数据类型进行比较，这种比较方法比较的是两边的值
				
				- 2.引用数据类型进行比较，这种方法比较的是两边的地址值
	
	- ## 逻辑运算符：
	
		 - `&`  与
		 
		 - `|`  或
		 
		 - `!`  非
		 
		 - `^`  异或：
		 
			 - 与或相反，两边都为 true 则返回 false，只有一边为 true 则返回 true
	
	- ## 短路逻辑运算符：
	
		- 逻辑运算符在 java 中执行效率较低，例如a = 1，判断 `a > 2 & a <10` 时，左边已经是false，但右边会继续进行判断，然后再对&进行判断，为了提高效率，有了短路逻辑运算符：
		
			- &&：
			
				- 短路与，左边为 false 时，右边不再执行
			
			- ||：
			
				- 短路或，左边为 true 时，右边不再执行
	
	- ## 三元运算符：
	
		- 可以理解为简单的 if 语句，具体语法为：
		
			- `关系表达式?表达式1:表达式2;`
			
			- `a>b?a:b;`
		
		- 如果关系表达式a>b的结果是true，则返回表达式1 a，否则返回表达式2 b
