- 概述：在网络通信协议下，不同计算机上运行的程序，进行的数据传输
- 常见软件架构：
	- c/s：Cilent/Server
		- 用户本地需要下载安装客户端程序，在远程有一个服务端程序
		- 优缺点：
			- 用户体验好
			- 需要同时开发客户端与服务端
			- 用户更新与下载较为复杂
	- b/s：Browser/Server
		- 通过浏览器访问不同网址，来访问不同的服务器
		- 优缺点：
			- 不需要开发客户端
			- 用户不需要下载，打开浏览器即可使用
			- 应用过大用户体验会受到影响
- 三要素：
	- IP：
		- 概述：
			- 设备在网络中的地址，是唯一标识
			- IPv4 只有 2^32 个，已经用完了
			- IPv6 有 2^128 个
			- 可以使用局域网 IP 解决 IP 不够的问题
			- 127.0.0.1 表示本机
		- InetAddress：
			- 概述：用来表示 IP 的类
			- 方法：
				- ![[InetAddress方法.png]]
	- 端口号：
		- 概述：应用程序在设备中的唯一标识，由两个字节表示的整数，取值为 0-65535
	- 协议：
		- 概述：数据在网络中传输的规则，常见有 UDP，TCP，Http，Https，ftp
		- UDP 协议：
			- 全程用户数据协议（User Datagram Protocol）
			- UDP 协议是面向无连接通信协议，速度快，但有大小限制，一次最多发送 64K，不安全，易丢失数据
			- 面向无连接指不管是否连接成功，都发送数据
		- TCP 协议：
			- 传输控制协议（Transmission Control Protocol）
			- TCP 协议书是面向连接的通信协议
			- 速度慢，但没有传输限制，数据安全
			- 确保连接成功后才会发送数据
- UDP 传输数据：
	- 发送：
		1. 创建发送端的 DatagramSocket 对象
			- 格式：`DatagramSocket ds = new DatagramSocket(int port);`
				- port 是 int 类型数据，代表发送数据使用的端口号
				- 如果不传入端口号，则在所有可用端口中随机选择一个可用的使用
		2. 数据打包
			- 格式：`DatagramPacket dp = new DatagramPacket();`
				- 传入参数：
					- `byte[] buf`：要发送的字节数组
					- `int length`：发送的字节数组长度，一般全部发送，也可以替换成起始索引+发送长度只发送部分
					- `InetAddress address`：要发送到哪个 ip
					- `int port`：发送数据使用的端口号
		3. 发送数据
			- 格式：`ds.send(dp);`
		4. 释放资源
			- 格式：`ds.close();`
	- 接收：
		1. 创建接收端的 DatagramSocket 对象
			- 接收的端口要与发送的一致
		2. 接收打包好的数据
			- 同样创建 DatagramPacket 对象，只传入空 byte 数组和 byte.length 即可
			- 调用`dp.receive(dp);`
				- 该方法是阻塞的，只有获取到消息后才会往下执行代码
		3. 解析数据
			- `getAddress`：获取发送者 ip
			- `getData`：获取发送来的数据				
			- `getPort`：获取端口
		4. 释放资源
			- 格式：`ds.close();`
- UDP 三种通信方式：
	- 单播： 一对一，一台电脑只给另一台电脑发送数据
	- 组播：
		- 一对多，一台电脑给一组电脑发送数据
		- 244.0.0.0 ~ 244.0.0.255 为预留的组播 ip
		- 代码实现：
			- 发送端
				- 不再创建 DatagramSocket 对象，而是创建 MulticastSocket 对象
				- 在对象 dp 中指定发送 ip 为 244.0.0.1 
				- 使用 MulticastSocket 对象的 send 方法发送
			- 接收端：
				- 不再创建 DatagramSocket 对象，而是创建 MulticastSocket 对象
				- 使用 `ms.joinGroup(InetAddress.getByName("244.0.0.1"))`添加到 244.0.0.1 这一组
	- 广播：
		- 给局域网中所有电脑发送数据
		- 写代码时将发送 ip 改为 255.255.255.255
- TCP 协议传输数据：
	- 概述：
		- TCP 在通信的两端各建立一个 Socket 对象，通信前要保证连接已经建立
		- 通过 Socket 产生的 I/O 流来进行网络通信
	- 客户端发送数据：
		1. 创建发送端的 Scoket 对象
			- `Socket socket = new Socket(String host,int port)`
				- host：接收端 ip
				- port：使用的端口号
				- 在创建 socket 对象时会尝试连接服务端，连接失败就会报错 
		2. 获取输出流，写出数据
			- 获取输出流：`OutputStream os = socket.getOutputStream();`
			- 写出数据：`os.write();` 只能写出字节数据
		3. 释放资源
			- `os.close; socket.close;`
	- 服务端接收数据
		1. 创建服务器接收端的 ServiceSocket 对象
			- `ServiceSocket ss = new SrrviceSocket(int port)`
				- port：使用的端口号
		2. 监听发送端连接，返回一个 Scoket 对象
			- `Socket socket = ss.accept();`
			- 如果一直没有客户端连接，代码会卡死在这里
		3. 获取输入流，读数据
			- `InputStream is = scoket.getInputStream();`
				- 不知道发送来的数据有多少，可以用流的循环读取来读取数据
				- 发送中文时因为默认编码是 utf-8，因此使用字节流解析会出错，需要更换为字符流`InputStreamReader isr = new InputStreamReader(is);`
		4. 释放资源
			- `socket.close; ss.close;`
- 三次握手和四次挥手
	- 三次握手：
		- 客户端向服务器发送连接请求等待服务器确认
		- 服务器返回一个响应，告诉客户端收到了请求
		- 客户端再次发出确认消息，连接建立
	- 四次挥手：
		- 客户端向服务端发送取消连接请求
		- 服务端向客户端返回一个响应，表示接受到请求
		- 服务端在处理完所有正在传输的数据后，向客户端发出确认取消信息
		- 客户端再次发送确认消息，连接关闭
