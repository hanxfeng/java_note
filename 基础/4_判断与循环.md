- # switch 语句：
	
	`switch(变量){`
		`case 1:`
			`语句;`
			`break;`
		`case 2:`
			`语句;`
			`break`
		`default:`
			`语句;`
			`break;`
	`}`
	
	- ## 注意：
		- 当所有 case 都与变量不匹配时，就会执行 default，default 不是必须的
		
		- break 需要加上，不然会出现 case 穿透现象，例如变量为 1，case1 的语句中没有 break,那么后续 case 中的语句也会依次执行。
		
		- case 值必须是确定的常量，switch 语句不支持 double 之类的浮点数类型！！！！
		支持 int，char，str，enum	
	
	- ## java12+版本的新语法：
		`switch(变量){`
			`case 1,2,3,4 -> {语句};`
			`case 5,6,7,8 -> {语句};`
			`default -> {语句};`
		`};`
	
	- ## 注意
		- 这种语法不会触发case穿透，但仅在较高版本的 java 中可用
		
		- 只有语句是单行可以这样写，多行还是要加大括号
		
	- ## yield 关键字：
		- 在 java13+ 中引入，作用是从 switch中 返回一个值
```java
		String name = switch (month) {
		    case 1, 2, 3 -> {
		        yield "春季";
			    }
		    case 4, 5, 6 -> {
		        yield "夏季";
			    }
		    case 7, 8, 9 -> {
		        yield "秋季";
			    }
		    case 10, 11, 12 -> {
		        yield "冬季";  
			    }  
		    default -> {
		        yield "输入错误";  
			    }  
			 };
```
		yield 与 return 的不同：
			yield: 只在 switch 表达式中使用，返回值给 switch 表达式
			return: 在方法中使用，返回值给方法调用者
		if 与 switch 的使用场景：
			一般讲 if 可以完全替代 switch,switch 相较于if将其优点只有阅读性更好，性能更高，因此在有限个 case 时使用 switch 而不是 if ,可以提高运行效率


- # for循环：
	
	`for (int i = ; i <= 100; i++){`
		`语句;`
	`}`
	
	- 增强 for 循环：
		如果中间语句是类似 `i<arr.legth` 这种的，而且后面语句中只需要用到数组内的值不需要用到索引时，可以使用增强for循环
		`for (int j : arr) {`
		`System.out.println(j);`
		`}`
		上面的代码会逐个输出数组 arr 中的元素
