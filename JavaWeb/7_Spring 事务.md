- Spring 事务：
	- 实现：在添加 @Transactional 注解，可以放在方法/类/接口上，代表将当前类/方法/接口交给 spring 进行事务管理
- 回滚：
	- 概述：默认情况下，只有出现 RuntimeException 才会回滚异常，roolbackFor 属性用于控制出现何种异常类型回滚事务，代码为：`@Transactional(rollbackFor = Exception.class)` 出现任何异常都回滚事务
- 事务传播行为：
	- 概述：当一个事务方法被另一个事务方法调用时，这个事务方法该如何进行控制

- AOP 基础：
	- 概述：面向切面编程，面向方面编程，面向特定方法编程，动态代理是面向切面编程最主流的实现，SpringAOP 是 Spring 框架的高级技术，旨在管理 bean 对象的过程中，主要通过底层的动态代理机制，对特定方法进行编程
	- 示例：![[AOP示例.png]]
	- 核心概念：
		- 连接点：
			- joinPoint 可以被 AOP 控制的方法（暗含方法执行时的信息），所有方法都是连接点。
		- 通知：
			- Advice，指重复执行的逻辑，也就是共性功能。示例中是统计方法运行时间
		- 切入点：
			- PointCut，指匹配连接点的条件，是一个表达式或规则，通知仅会在切入点方法被执行时应用，通知仅会在切入点方法被执行时应用。切入点是指匹配 @Around 属性值的方法，示例中是 service 这个包下的所有方法
		- 切面：
			- Aspect，通知与切入点表达式加在一起就是切面，带有 @Around 的类就是切面类
		* 目标对象：
			* Target，通知应用的对象就是目标对象，切入点表达式匹配的方法就是目标对象
	- AOP 执行流程：
		-  程序运行时，会基于动态代理技术为目标对象创建一个代理对象，在代理对象中实现通知和方法的原始功能，程序运行注入的不再是目标对象而是代理对象
	- 通知类型：
		- @Around：环绕通知，此注解标注的通知方法在目标方法前后都执行
		- @Before：前置通知，此注解标注的通知方法在目标方法前执行
		- @After，后置通知，此注解标注的通知方法在目标方法后执行
		- @AfterReturning，返回后通知，此注解标注的方法在目标方法后被执行，有异常不执行
		- @AfterThrowing，异常后通知，此注解标注的方法发送异常后执行
		- 除了 @Around 以外，所有通知类型仅写通知即可，不需要引入 ProceedingJoinPoint ，不需要在代码中调用原始方法运行，例：![[非Around通知类型代码示例.png]]
		- @Around 的返回值必须指定为 Object，来接收原始方法的返回值
		- @PointCut 可以将重复的切入点表达式抽取成一个方法，之后的通知调用这个方法即可，不需要再写切入点表达式，方法标识符设置为 public 后当前包下的所有 AOP 类都可以使用，例：![[PointCut使用示例.png]]
	- 通知顺序：
		- 在原始方法执行之前执行的通知按 A-Z 的顺序执行，在原始方法执行之后执行的通知按 Z-A 的顺序执行
		- 可以使用 @Order(数字) 来控制执行顺序， 在原始方法执行之前执行的通知数字小的先执行，在原始方法执行之后执行的通知数字大的先执行
	- 切入点表达式 execution：
		- execution：
			- `execution(访问修饰符? 返回值 包名.类名.?方法名(方法参数) throws 异常?`
				- 带有 ? 为可省略部分，包括访问修饰符，包名类名，异常
			- 通配符：
				- `*` ：单个独立的任意符号，可以通配返回值、包名类名方法名、任意类型的一个参数，也可统配包名类名方法名的一部分
				- `..`：多个连续的任意符号，可以通配任意层级的包，或任意类型，任意个数的参数
				- 可以使用 与（&&），或（||），非（！），组成较为复杂的表达式，`execution(表达式1) || execution(表达式2)`
		- @annotation：
			- 匹配标识有特定注解的方法。需要自定义一个注解 ：![[annotation注解示例.png]]
			- 然后在需要切入的方法上加上 @MyLog 这个注解，在@annotation的属性值中指定注解的全类名即可`@Before("@annotation(com.example.java_practice.MyLog)")`
	- 连接点：
		- Spring 中使用 JoinPoint 抽象了连接点，用它可以获取方法执行的信息，如目标类名，方法名，方法参数等。
			- 对于 @Around 通知，获取连接点信息只能使用 ProceedingJoinPoint
			- 对于其他四种通知，获取连接点只能使用 JoinPoint 它是 ProceedingJoinPoint 的父类型
