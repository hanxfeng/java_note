- HTTP 协议：
	- 概述：中文名 超文本传输协议，规定了浏览器和服务器之间数据传输的规则
	- 特定：
		- 基于 TCP 协议，面向连接，安全
		- 基于请求-响应模型，一次请求对应一次响应
		- HTTP 协议是无状态协议，对于事务处理没有记忆能力，每次请求-响应都是独立的。
			- 缺点：多次请求间不能共享数据
			- 优点：传输速度快
	- 请求协议：
		- ![[请求数据格式示例.png]]
		- 第一行：请求行，格式：`请求方式 资源路径 协议`
			- 请求方式常见的是 GET 和 POST，也有其他
			- 资源路径指网址
			- 协议指传输方式，常见 http，https
		- 第二行：请求头，格式：`key:value` 一个键对应一个值
			- 常见请求头键值对含义：![[请求体含义.png]]
		- 第三行：请求体，POST 请求专有，存放请求参数
	- 响应协议：
		- ![[响应数据格式示例.png]]
		- 第一行：响应行，格式：`协议 状态码 描述`
			- 协议指传输方式，常见 http，https
			- 状态码： 可大体分为5类，常见有 200 成功，404 无响应等
				- 常见响应码及其含义：![[响应码含义.png]]
			- 描述：返回的描述
		- 第二行：响应体，格式：`key:value` 一个键对应一个值
			- 常见响应头键值对含义：![[常见响应头.png]]
		- 第三行：响应体：存放响应数据
