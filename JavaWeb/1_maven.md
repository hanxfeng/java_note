- 概述：用于管理和构建 java 项目的开源工具
- 作用：
	1. 进行依赖管理：方便快捷的管理项目依赖的资源（jar包），避免版本冲突问题
	2. 统一项目结构：
		- 提供标准、统一的项目结构
		- ![[标准maven项目结构.png]]
	3. 项目构建：标准跨平台的自动化项目构建
- 依赖管理：
	- 依赖配置：
		- 需要先书写`<dependependencies> 和 </dependependencies>` 在中间书写包相关
		- 再书写`<dependency> 和 </dependency>` 在中间书写导入包的相关信息
		- `<groupId>` 指组织 Id，即开发这个包的组织网址
		- `<artifactId>` 指这个包的具体名称
		- `<version>` 指这个包的版本
		- ![[maven导入示例.png]]
	- 依赖传递：
		- 依赖具有传递性
			- 如果一个包需要依赖另一个包，那么导入这个项目的时候会同时导入另一个包
		- 排除依赖：
			- 当有依赖不需要时，可以使用`<exclusions>` 排除依赖
			- 先书写`<exclusion>` 再在里面书写包的创建者和名称，不需要写版本，这个依赖就会被排除
			- ![[排除依赖.png]]
	- 依赖范围：
		- 默认情况：
			- 在默认情况下，导入的包会在 main 文件夹、test 文件夹和 package 指令范围内可用
		- scope 指定：
			- 可用通过在 version 后增加一个 scope 来指定包的生效范围，只能书写固定参数
			- ![[依赖范围控制.png]]
