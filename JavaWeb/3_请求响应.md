- 请求：
	- 简单参数：
		- 参数名和形参变量相同，定义形参即可接收参数
		- ![[Spring接收简单参数.png]]
			- 在代码中定义好参数类型和名称，接收参数时就可以自动转换
			- `@RequestMapping` ：是`GetMapping/PostMapping` 的普遍版本，支持接收多种请求类型的参数，`GetMapping/PostMapping`只能接受对应请求的参数
			- 如果传递参数和形参不同，可以设定`@RequestParam(name = "")` 注解,示例如下：![[RequestParma示例.png]]
				- required 参数为该参数是否为必须参数，默认为true 如果不填，在传入参数时没有该参数会返回400错误，无注解参数可以不传入，此时该参数的值为 null
		- 传统方式（了解即可）：![[传统接收参数方式.png]]
	- 简单实体参数：
		- 请求参数名与形参名数据类型相同即可，需要定义**实体类**进行接收![[简单实体参数代码示例.png]]
			- User 中需要包含传入的各项参数，如果传入的参数中确实某一个，该变量的值为 null
	- 复杂实体参数：在一个实体类中还有一个实体类
		- 在使用 Get 传入参数时，传入 `实体类.实体类变量 = value` 
		- 例：`GET http://localhost:8080/add?number.number1=1&number.number2=3`
	- 数组集合参数：
		- 数组参数：
			- 概述：请求参数名与形参数组名相同且请求参数为多个，定义数组类型形参来接收参数
			- 请求示例：`http://localhost:8080/add?number=1&number=3`
			- 服务端代码示例：![[数组参数服务端代码示例.png]]
		- 集合参数：
			- 概述：
				- 请求参数名与形参集合名形同且请求参数多个，使用`@RequestParam`绑定，与数组参数的不同在于数据传入到服务器时保持的数据类型不同
			- 请求示例：`GET http://localhost:8080/add?number=1&number=3`
			- 示例代码：![[集合参数代码示例.png]]
		- 日期参数：
			- 概述：
				- 因为时间格式有多种，所有需要使用`@DateTimeFormat`注解指定传入的时间格式
			- 请求示例：`GET http://localhost:8080/time?date=2026/2/21 12:00:33`
			- 示例代码：![[时间参数代码示例.png]]
		- json 参数：
			- 概述：json 数据键名与形参对象属性名相同，需要使用`@RequestBody`注解，然后定义实体类来接收
			- 请求：使用 POST 进行请求
			- 示例代码：![[jso参数示例代码.png]]
		- 路径参数：
			- 概述：通过请求 url 直接传递参数，使用 `{变量名}` 标识该路径，使用`@PathVariable`获取参数
			- 请求示例：`GET http://localhost:8080/id/1`
			- 示例代码：![[路径参数示例代码.png]]
- 响应：
	- `@ResponseBody`
		- 类型：方法注解，类注解
		- 位置：Controller 方法/类上
		- 作用：将方法返回值直接响应，如果返回值是 实体类/集合，将转为 json 后响应
		- 说明：较少单独使用，一般使用`@RestController`，`@RestController = @ResponseBody + @Controller`
	- 统一响应结果：
		- 需要自行构造 Result 实体类，至少定义 code，message，data 三个变量，构造方法则根据需求自行设置
		- 最简单的 Result 类代码示例：
		- ![[Result类代码示例.png]]
			- @Data 已经定义了 get set toString 方法
