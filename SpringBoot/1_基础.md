	- 概述：REST 风格是一种约定，用于简化网址书写和隐藏资源的访问行为，即无法通过地址得知是何种操作
	- 示例：![[REST风格示例.png]]
- yaml 格式：
	- 要求
		- 有 yml 和yaml 两种后缀
		- 对大小写敏感
		- 使用缩进表示层级关系，但不允许使用 tab 来缩进，只能使用空格缩进
		- 属性名与属性值之间使用 `: ` 冒号+空格分隔
		- `#` 表示注释
		- 同层级中属性名不允许重复
		- 使用 `- ` 减号+空格表示数据属于一个组中：
			- ![[yml格式示例2.png]]
	- 示例：
	- ![[yml格式示例.png]]
	- 读取：
		- 使用 @value 注解可以在配置文件（application.properties/yaml/yml）中读取数据，示例：![[读取数据示例1.png]]
			- 可以读取到属性名为 name 的数据
			- 多层级读取，使用 . 连接即可，例如 user.name 可以读取到 user 中的 name 的值
			- 读取数组中的数据：`name[0]` ，数组索引从 0 开始
	- 变量引用：
		- 使用 `${属性名}`，示例：![[变量引用示例.png]]
			- user 的值实际为 `li/li`
		- 如果需要使用转义字符，需要在属性值上加双引号变成字符串
