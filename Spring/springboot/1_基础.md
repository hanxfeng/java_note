- # starter
	- starter 是一种特殊的依赖管理描述符，主要作用有：
		1. 只要引入一个 starter 就可以获得该功能相关的全部依赖
		2. spring 会根据 starter 自动配置相关的 bean
	示例：
```java
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>
```

- # 配置文件
	- spring 的配置文件格式有三种：application.properties、application.yaml 和 application.yml
	
	- ## properties
		- 数据通过键值对的方式储存，例：`server.port = 8080`
		
	- 优先级 application.properties > application.yml > application.yaml
	- 不同配置文件中的相同内容按优先级进行覆盖，不同内容保留，假设三个文件都存在，都配置了 port，那么最终使用的是 properties 文件中的设置，只有 yaml 中设置了 port,那就使用 yaml 中的设置


- # yaml 格式：
	- ## 要求
		- 有 yml 和yaml 两种后缀
		- 对大小写敏感
		- 使用缩进表示层级关系，但不允许使用 tab 来缩进，只能使用空格缩进
		- 属性名与属性值之间使用 `: ` 冒号+空格分隔
		- `#` 表示注释
		- 同层级中属性名不允许重复
		- 使用 `- `减号+空格表示数据属于一个组中：
			- ![[yml格式示例2.png]]
			- 也可以使用`like: [game, sleep]` 这种方式来写简单数组
	
	- ## 示例：
	- ![[yml格式示例.png]]
	
	- ## 读取：
		- 使用 @value 注解可以在配置文件（application.properties/yaml/yml）中读取数据，示例：![[读取数据示例1.png]]
			- 可以读取到属性名为 name 的数据
			- 多层级读取，使用 . 连接即可，例如 user.name 可以读取到 user 中的 name 的值
			- 读取数组中的数据：`name[0]` ，数组索引从 0 开始
	
	- ## 引用单个数据：
		- 使用 `${属性名}`，示例：
		- ![[变量引用示例.png]]
			- user 的值实际为 `li/li`
			
		- 如果需要使用转义字符，需要在属性值上加双引号变成字符串
	
	- ## 引用全部数据：
		- spring 中有一个数据类型 Environment，可以用来读取配置文件中的全部数据，使用时需要加 @Autowired 注解自动注入
		
		- 示例：
	```
	@Autowired
	private Environment env;
	
	System.out.println(env.getProerty("user.name"))
	System.out.println(env.getProerty("like[0]"))
	```
	- ## 引用部分数据
		- 需要先定义一个 java 类来保存数据，然后通过注解的形式来读取数据
		```java
		# 假设yaml文件中存在user.id,user.userName,user.password数据
		@Component # 用于将类标记为spring管理的java bean 类
		@@ConfigurationProperties(prefix="user") # 确定数据所在位置
		@Data
		public class User {
			private int id;
			private String userName;
			private String password;
		}
		```


- # spring 整合 JUnit
	- ## 概述：
		- Junit 是 java 语言中用于进行测试的框架通过 @Test 等注解和方法可以快速频繁的验证代码正确性。
	
	- ## 整合
		- Spring Boot 整合 JUnit 主要依靠 `@SpringBootTest` 注解。 加上该注解后，Spring 会为这个测试类启动 Spring 容器，并自动完成依赖注入等准备工作。只需要用 `@Autowired` 注入要测试的组件，然后在测试方法上加上 `@Test` 注解（JUnit 提供），就可以快速进行测试。


- # Spring 整合 MyBatis
	- ## 概述：
		- MyBatis 封装了 JDBC 的复杂操作，可以使用更简单的方式与数据库进行交互，现在更多使用 MyBatis-PLUS，在不改变 MyBatis 核心能力的前提下简化了操作。
	
	- ## 整合
		1. 在 `pom.xml` 中添加 `mybatis-plus-boot-starter` 依赖
		
		2. 在配置文件中添加数据库连接信息（包括驱动类名，连接地址，连接账户和密码）
		
		3. - 在 Mapper 接口上添加 `@Mapper` 注解，或是在启动类上使用 `@MapperScan` 注解指定扫描路径，之后就能使用了。


- # Druid 数据库连接池
    - ## 概述：
        - Druid 是阿里巴巴开源的一款**为监控而生**的数据库连接池，在 Spring Boot 项目中负责管理和复用数据库连接。
            
        - 核心能力：**连接管理**（避免频繁创建/销毁连接）、**SQL 监控**（统计慢查询、执行次数等）、**SQL 防火墙**（防注入）、**连接泄漏检测**等。
            
        - 相比默认的 HikariCP，Druid 功能更丰富，尤其适合需要深度监控的生产环境。
    
    - ## 整合 Spring Boot
        
        1. 在 `pom.xml` 中添加 `druid-spring-boot-starter` 依赖（注意 Spring Boot 2 和 3 的 starter 不同）。
        
        2. 在配置文件中指定数据源类型为 Druid，并配置连接池参数（如 `initial-size`、`max-active`）以及监控相关配置（如 `stat`、`wall` 过滤器）。
        
        3. 启用监控页面（`stat-view-servlet` 配置），访问 `http://{ip}:{port}/{context-path}/druid` 可查看 SQL 执行情况。


- # 不同层的框架搭建
	- ## Entity 层：
		1. 使用 private 定义实体类中的变量
		2. 在类上加 @Data 注解
	
	- ## Mapper 层：
		1. 继承 `BaseMapper<T>`
		2. 在类上加注解 @Mapper
	
	- ## Service 层：
		1. 创建类时继承 `IService<T> `
		
		### ServiceImpl 层 ：
		1. 继承 `ServiceImpl<T Mapper,T>
		2. 实现对应的 Service
		3. 在类上加注解 @Service
	
	- ## Controller 层：
		1. 在类上加 @RestController 注解
		2. 在类上加 @RequestMapping("/") 注解，并在 “” 中写上其对应的接口


- # 额外添加的功能
	- ## 分页查询：
		- 在 Mybatis-PLUS 中，实现分页查询需要自定义一个拦截器，格式较为固定
		
		```java
		@Configuration  
		public class MybatisPlusConfig {  
		    @Bean  
		    public MybatisPlusInterceptor mybatisPlusInterceptor() {  
		        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();  
		        interceptor.addInnerInterceptor(new PaginationInnerInterceptor());  
		        return interceptor;  
		    }  
		}
		``` 
	- ## 统一返回值：
		- 为了方便前端对返回的数据进行解析，要统一返回数据的格式，需要定义一个实体类 R，其中有三个变量 ：
			- code        ：编码，1成功，0和其它数字为失败
			- message  ：错误信息
			- data         ：返回前端的数据
		
		- 两个静态方法：
			- success  ：执行成功时返回给前端的内容
			- error      ：执行失败时返回给前端的内容
		
		- 一下是一个示例：
			```java
			@Data  
			@NoArgsConstructor  
			@AllArgsConstructor  
			public class Result<T> implements Serializable {  
				  
			    private Integer code; //编码：1成功，0和其它数字为失败  
				  
			    private String msg; //错误信息  
				  
			    private T data; //数据  
				  
			    public static <T> Result<T> success(T object) {  
			        Result<T> r = new Result<T>();  
			        r.data = object;  
			        r.code = 1;  
			        return r;  
			    }  
				  
			    public static <T> Result<T> error(String msg) {  
			        Result r = new Result();  
			        r.msg = msg;  
			        r.code = 0;  
			        return r;  
			    }  	  
			}
			```
