- # 概述：
	- Spring Cache 是一个框架，实现了基于注解的缓存功能，只需要简单加一个注解，就能实现缓存功能
	
	- Spring Cache 提供了一层抽象，可以通过 CacheManager 接口来使用不同的缓存技术：
		- Redis                               :   RedisCacheManager
		- EhCache                          :   EhCaCheCacheManager
		- Google 的 GuavaCache  ： GuavaCacheManager


- # 常用注解：
	
	- @EnableCaching  ：开启缓存注解功能
	- @Cacheable         ：在方法执行前 spring 先查看缓存中是否有数据，如果有直接返回缓                                   存数据，如果没有，调用方法并将方法返回值放入缓存中
	- @Cacheput          ：将方法的返回值放入缓存中
	- @CacheEvict        ：将一条或多条数据从缓存中删除


- # spring 项目使用：
	
	1. 导入Maven坐标：
		spring-boot-starter-data-redis   、   spring-boot-starter-cache
	2. 配置配置文件
		示例：
		spring.redis.host=111.222.333.444  
		spring.redis.port=6379
		spring.cache.redis.time-to-live = 1800000 # 配置缓存过期时间，单位是毫秒
	3. 在启动类上加注解 @EnableCaching 开启缓存注解功能
	4. 在 Controller 的方法上加入相关注解


- # @Cacheput 注解：
	
	- 在 @Cacheput 中有两个关键参数， value 和 key 和 Redis 数据库中不同， value 是指缓存名称，每个 value 下可以有多个 key，key 是指缓存的 key，value 与 key 会按照一定方式连接，成为Redis数据库中的key，例如 value::key 的形式
	
	- key 可以使用特殊方式获取值，例如#resulr 代表返回值，#user 在函数的形参为 user 时代表输入函数的值，可以使用#user.id，来获取user中的id的值
	
	- 可以通过#root.args[0]，#形参名，#p0，三种方式来获取实参的值，0 指形参是第几个形参，第一个就是0，第二个就是1
	
	- 示例：@Cacheput(value = "userName", key = "#user.id")


- # @CacheEvict 注解： 
	
	- 与 @Cacheput 相同，@CacheEvict 也有两个同样的参数，起同样的作用。@CacheEvict用于将对应数据从缓存中删除
	
	- allEntries 参数，默认为 False，当为 True 时，删除所有缓存数据


- # @Cacheable 注解：
	
	-  与 @Cacheput 相同，@Cacheable 也有两个同样的参数，起同样的作用。@Cacheable 用于在查询数据时，如果数据不存在于缓存中，则查询数据并将数据缓存，如果数据存在于缓存中，则直接返回缓存中的数据
	
	- @Cacheable 还有两个参数 condition 和 unless ，condition 表示满足什么条件后缓存数据，例如不为 null （condition = "#user != null") ，unless 表示不满足什么条件后缓存数据（unless = "#result == null")
	- 
	- 需要注意的是，condition 不能使用#result 方法，如果要使用#result 则需要使用相反的条件的 unless 方法，具体可以使用的方法可以查询源码
