- 概述：
	- java 将其使用的内存分为5个部分，分别是：
		- 栈：方法运行时使用的内存，比如 main 方法运行，进入方法栈中执行
		- 堆：储存对象或数组，new 创建的，都储存在堆内存中
		- 方法区/元空间：储存类（class）的结构信息（包括类的元数据、常量池、静态变量等）
		- 本地方法栈：jvm 操作系统功能时使用
		- 寄存器/程序计数器：给 cpu 使用
- 基本数据类型与引用数据类型的内存原理：
	- 基本数据类型：
		- 包括整数类型，浮点数类型，布尔类型和字符类型
		- 对于基本数据类型来说，在 main 方法进入栈内存后，如果定义了一个基本数据类型，例如 `int a = 10;` 那么就会在栈中开辟一块内存空间来储存 a = 10 这段信息
	- 引用数据类型：
		- 包括除基本数据类型外的所有数据类型
		- 与基本数据类型不同，引用数据类型由于需要使用 new 关键字进行创建，因此储存在堆内存中，而栈内存储存的是该数据在堆内存中的地址值，因此直接打印数组得到的是地址值而不是数组本身。
		- 如果两个变量指向了同一个引用数据，那么一个变量对这个数据进行修改，另一个也会改变，因为两者不是单独储存，而是通过地址值连接到了同一个堆内存中的数据
- 方法的内存原理：
	- 方法进出栈类似将子弹压进弹夹，先进后出，方法按照代码上的顺序依次进出栈中，后进先出，这里进指的是方法被调用后进入栈中，出指的是方法执行结束后从栈中释放
- this 的内存原理
	- this 作用：区分成员变量和局部变量，this 引用的是成员变量
	- this 本质：代表方法调用者的地址值
	- this 举例理解，使用 new 新建方法 s ，方法 s 中有函数 `setA(a)`：`this.a = a` 那么，当调用`s.setA(3)` 时，this 代表方法调用者 s 的地址值，这里假设是 001 ，a 是离它最近的变量，这里是传入的形参 3 ，这段代码在内存中就是将 3 赋值给地址 001 即 s 的 a。
- 成员变量与局部变量的区别

| 区别   | 成员变量    | 局部变量          |
| :--- | :------ | :------------ |
| 类中位置 | 类中方法外   | 方法内，形参        |
| 初始化值 | 有默认初始化值 | 无初始化值，使用前需要赋值 |
| 内存位置 | 堆内存     | 栈内存           |
| 生命周期 | 随对象存在   | 随方法存在         |
| 作用域  | 整个类中有效  | 当前方法中有效       |

- 字符串内存原理：
	- 直接赋值会复用字符串常量池（位于堆内存中，是一个特殊区域，专门存放字符串）中的
	- 使用 new 关键字创建的不会复用，而是存放在堆中
	
	- 字符串拼接底层原理：
		- 没有变量参与的拼接：
			- 如果拼接过程没有变量参与，如 `"abc = a"+"b"+"c"` ，在编译时有特殊优化,直接就是最终结果 `abc = "abc"`
		- 有变量参与：
			- jdk8 以上版本会预估拼接后字符串长短，然后创建一个对应长度的数组，最后将这个数组再转为字符串
		- 因此在多个字符串拼接时会创建多个对象，浪费性能，需要进行拼接时使用 StringBuilder 进行以提高效率
	- StringBuilder 提高效率的原理：
		- 它会创建一个对象来容纳字符串，进行拼接时就向这个对象中添加字符串，不再创建新对象

- static 修饰的变量的内存原理：
	- 如果一个类中存在被 static 修饰的变量，那么在加载这些变量的时候，会在堆内存中开辟一个位置：静态储存区，来专门保存这些静态变量，默认值是null/
	- 静态变量随着类加载而加载，先于对象出现''

- 继承的内存原理：
	- 执行顺序：
		 Java程序执行时的内存流程
			 1. 类加载阶段（方法区/元空间）
			- 程序启动时，JVM首先加载包含 `main` 方法的类到方法区
			- 如果该类继承自父类，会先加载父类，再加载子类（不是方法，是整个类的结构信息）
			- 方法区存储的是类的模板信息（包括方法代码、常量、静态变量等）
			 2. 方法执行阶段（栈内存）
			- `main` 方法开始执行，JVM为其创建一个栈帧压入栈中
			- `main` 方法是程序的起点，但和其他方法一样，都是在栈中创建栈帧执行
			- 所有局部变量（包括对象引用）都存储在栈帧的局部变量表中
			 3. 对象创建阶段（堆内存）
			- 遇到 `new` 关键字时，在堆内存中分配空间创建对象
			- 对象包含所有继承来的成员变量（父类+子类）
			- 栈中的引用变量存储对象在堆内存中的地址值
- 多态内存原理：
	- 变量：
		- 在使用多态创建对象时，会在堆内存创建一个空间储存变量，这个空间有两部分，一部分储存父变量，一部分储存子变量，在使用变量时查询的是父变量，因此会出现编译看左边运行看左边的现象
	- 方法：
		- 因为子类重写方法后会替代虚方法表中父类的方法，因此在执行时会执行子类的方法，而在编译时，检查的是父类的虚方法表，所以会出现编译看左边运行看右边的现象
	- 优势：方法中使用父对象作为参数，可以接受所有子对象
	- 弊端：不能使用子对象的特有功能，可以通过强转为子类类型来解决
- 内部类的内存原理：
	- 外部类与内部类是两个不同的 class 文件
	- 在使用 `外部类名.内部类名 对象名 = new 外部类对象名().new 内部类对象名();` 创建内部类对象时，有一块储存外部类对象的内存，一块储存内部类对象的内存，内部类对象中有一个隐藏的 `外部类名.this` ，地址值是外部类对象的储存位置，对象所报存的地址值是内部类对象的地址。使用 `外部类名.this.变量名` 调用的是外部类的变量，因为 `外部类名.this` 指向的是外部类的地址值
